### 旅行日程自動作成Webサービス新規開発プロジェクトのAPI開発/SPA開発【Rails/Angular4/TypeScript】（2017年7月〜2018年9月）

【プロジェクト概要】   
インバウンド旅行者向け、国内旅行者向けの2つの旅行日程自動作成アプリケーションのバックエンド、フロントエンド開発。

【担当業務】   
- ユーザー画面と管理画面のそれぞれにおいて以下の開発を担当。
- Railsを用いたAPI実装、Angular4を用いたSPA実装。
  - 画像アップロード、ページング、名称検索機能など
  - 外部API（Facebook、GoogleMaps、楽天API、JCB、駅探API）との連携。  
　　※Facebookについては、新規会員登録・ログイン・シェア機能で利用。
  - 観光地スポットデータ（数千件）のDBへの初期データ投入ロジック実装。
  - SessionStorage、LocalStorageを利用した実装。  
　　※ログイン、ログアウト処理実装内でのLoginTokenの取り扱い、別タブでの処理、Component間の値の受け渡し（新規作成画面、編集画面、シェア画面のComponent間）等で使用。
- （手動による）単体テスト、シナリオテストのテストケース作成と実施。
- ステージング、本番環境デプロイ。

※両サービスの違い  
インバウンド向けサービス：（海外からの旅行者が）メインの移動手段として飛行機を利用するのを前提に作成。  
国内旅行者向けサービス：メインで車を利用するのを前提に作成。
そのため、国内旅行者向けサービス開発時は、上記事項に加えて以下の対応を行った。  
- API新規追加・改修。
- 商業施設・ガソリンスタンドのデータのDBへの初期データ投入ロジック実装。
- 本番、ステージング環境構築（Nginx、DB、pumaの設定）。
- 管理画面のCRUD機能実装（商業施設・ガソリンスタンドのデータ）。
- 観光地スポットデータ（数千件）のDBデータ更新作業。
- データ更新後のDBのデータが正しく入っているか、画像ディレクトリと比較して不足しているものがないかチェックするツールを実装し、調査、調整作業実施。

【発揮したバリュー】  
- パートナー企業の開発者やデザイナー、ディレクターの方と連携し開発を進めることができたこと。
- （受託開発のため）顧客と直にコミュニケーションを取りながら要件を整理し開発を行うことができたこと。
